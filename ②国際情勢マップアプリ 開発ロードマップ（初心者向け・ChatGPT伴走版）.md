# 国際情勢マップアプリ 開発ロードマップ（初心者向け・ChatGPT伴走版）

## 1. このロードマップの目的
このロードマップは、**プログラミング初心者が途中で迷わず、1つずつ確実に完成させるための実行計画**です。  

仕様書は「何を作るか」を定義する文書ですが、このロードマップは「どういう順番で、どう進めれば完成しやすいか」を示す文書です。

このプロジェクトで一番大事なのは、最初から完璧を目指さず、**小さく動くものを積み上げること**です。

---

## 2. この開発で絶対に守る方針

### 方針1：一度に1つの機能しか作らない
たとえば、地図表示が終わっていないのにCSV読込や詳細表示を同時にやらない。  
必ず順番に進める。

### 方針2：毎回「動く状態」を残す
コードを大きく変えすぎず、**前より悪くしない**ことを重視する。

### 方針3：エラーが出たら、自己流で広げずに切り分ける
初心者開発で一番危険なのは、問題が1つなのに3か所同時に直してしまうこと。  
必ず原因を小さく切り分ける。

### 方針4：ChatGPTには「今どの段階か」を明示して聞く
「今はSTEP3です」「地図は表示済み、国クリックで止まっています」のように伝えると、かなり正確に支援を受けやすくなる。

## 3. 全体スケジュール像
このアプリは、初心者が無理なく進めるなら、次の6フェーズに分けるのが最も安全です。

## 4. フェーズ1：準備フェーズ

### 目的
開発環境と必要ファイルの形を整える。  
ここではまだ高機能なことはしない。

### このフェーズでやること
- Node.js をインストール
- VS Code をインストール
- React + Vite プロジェクトを作る
- アプリを起動してブラウザ表示確認
- Git を入れる（できれば）
- プロジェクトのフォルダ構成を整える

### 完成条件
- `npm run dev` で画面が開く
- Reactアプリがローカルで動く
- プロジェクトの場所が自分で把握できている

### 初心者が詰まりやすい点
- Node.jsの入れ忘れ
- ターミナルの場所が違う
- フォルダ階層がわからない

### ChatGPTに聞くときの例
```text
今は準備フェーズです。
React + Vite の初期セットアップ中です。
Windowsで、VS Codeを使っています。
どのフォルダで何のコマンドを打つか、初心者向けに具体的に教えてください。
```

---

## 5. フェーズ2：土台構築フェーズ

### 目的
まずは「アプリの骨組み」を作る。  
この段階では**地図が表示されれば勝ち**です。

### このフェーズでやること
- Leaflet / React Leaflet の導入
- 世界地図の表示
- CSSの最低限の調整
- 画面を「地図エリア」「右側パネル」に分ける

### この段階ではまだやらないこと
- 国クリック
- CSV読込
- 航路表示
- 検索

### 完成条件
- 世界地図が画面に表示される
- 地図が崩れず表示される
- 地図の横に詳細パネルの枠だけ表示される

### 成果物
- `MapView.jsx`
- `DetailPanel.jsx`
- `App.jsx`
- `App.css`

### 初心者が詰まりやすい点
- LeafletのCSS読み込み忘れ
- 地図の高さが0で表示されない
- `MapContainer` の親要素に高さがない

### ChatGPTに聞くときの例
```text
今は土台構築フェーズです。
React Leaflet で世界地図を表示したいです。
初心者向けに、必要なインストールコマンド、作るファイル、コピペできるコードを順番に教えてください。
```

---

## 6. フェーズ3：地図操作フェーズ

### 目的
地図を「ただ見るもの」から「操作できるもの」に変える。

### このフェーズでやること
- 国境GeoJSONを読み込む
- 各国をクリック可能にする
- クリックした国名またはISOコードを取得する
- 選択中の国をハイライトする
- 右パネルに仮データを表示する

### 完成条件
- 国をクリックできる
- どの国をクリックしたか分かる
- 右パネルにその国の名前が出る
- 選択中の国の色が変わる

### 成果物
- `countries.geojson`
- `mapHelpers.js`
- `MapView.jsx` 改良版
- `DetailPanel.jsx` 仮表示版

### 初心者が詰まりやすい点
- GeoJSONの属性名が想定と違う
- ISOコードの場所が分からない
- クリックイベントの書き方が分からない

### この段階で大事な考え方
まだCSVとつながなくていいです。  
まずは「クリックすると国が取れる」だけに集中します。

### ChatGPTに聞くときの例
```text
今は地図操作フェーズです。
世界地図は表示済みです。
次に、GeoJSONの国をクリックして、右パネルに国名を出したいです。
App.jsx と MapView.jsx のどこをどう直すか、初心者向けに具体的に教えてください。
```

## 7. フェーズ4：データ連携フェーズ

### 目的
地図クリックとニュースデータをつなぐ。  
ここで初めて「アプリっぽさ」が出てきます。

### このフェーズでやること
- `events.csv` を作成
- CSVを読み込む
- `country_iso3` でGeoJSONの国コードと照合する
- クリックした国に対応するニュースを取得する
- 詳細パネルに実データを表示する

### 完成条件
- CSVを更新すると表示内容が変わる
- 国をクリックすると、その国の出来事が出る
- タイトル、概略、関係国、影響が表示される

### 成果物
- `events.csv`
- `loadCsv.js`
- `App.jsx` データ連携版
- `DetailPanel.jsx` 実データ版

### 初心者が詰まりやすい点
- CSVの置き場所
- fetchパスの間違い
- CSV列名のスペル違い
- ISOコードの不一致

### この段階での重要ルール
CSVの列名は途中で変えない。  
ここを変えると、あとで全部壊れやすくなります。

### ChatGPTに聞くときの例
```text
今はデータ連携フェーズです。
国クリックまではできています。
次に、public/data/events.csv を読み込んで、クリックした国のISO3コードと一致するデータを右パネルに出したいです。
どのファイルを新規作成し、どこに何を書くか、初心者向けに具体的に教えてください。
```

### 完成条件
- 情報が読みやすい
- 航路表示を切り替えられる
- 見た目が最低限整っている

### 成果物
- `routes.geojson`
- `LayerToggle.jsx`
- `RouteLayer.jsx`
- `DetailPanel.jsx` 改良版

### 初心者が詰まりやすい点
- レイヤーの重なり順
- トグル状態管理
- 航路データの座標形式

### ChatGPTに聞くときの例

次に航路レイヤーを追加して、チェックボックスでON/OFFしたいです。
初心者向けに、追加するファイルと修正箇所を具体的に教えてください。

## 12. 初心者がやってはいけない進め方

### NG1：いきなり地球儀に行く
最初に難しいものへ行くと、進捗が止まりやすい。

### NG2：DBや管理画面を先に作る
今の段階では不要。まずはCSVで十分。

### NG3：見た目を早く作り込みすぎる
機能が不安定なままUIを触ると、あとでやり直しになる。

### NG4：エラーを要約してChatGPTに渡す
エラーは必ず全文貼る。省略すると原因特定が難しくなる。

### NG5：複数の回答を混ぜて同時に直す
1回の修正で1つの方針に絞るほうが安全。

## 13. ChatGPTへの相談ルール（このプロジェクト専用）
今後ChatGPTに聞くときは、毎回以下の順で伝えると話が早いです。

### 必須で書くこと
1. 今どのフェーズか
2. 何ができているか
3. 何ができていないか
4. 何をしたいか
5. エラー全文
6. 関連ファイル全文または必要部分

### コピペ用テンプレート
```text
私はプログラミング初心者です。
外注なしで、自分で国際情勢マップアプリを作っています。
コード作成はChatGPTに支援してもらう前提です。

技術構成は React + Vite + React Leaflet です。
CSVでニュースデータを管理し、国クリックで詳細を表示するアプリを作っています。
航路レイヤーのON/OFFも入れたいです。

今のフェーズ：
（例：フェーズ4 データ連携フェーズ）

今できていること：
（例：世界地図表示、国クリック、国名表示まで完了）

今困っていること：
（例：events.csv を読み込んでも右パネルに何も出ない）

やりたいこと：
（例：クリックした国のISO3コードと一致するCSVデータを表示したい）

ファイル構成：
（必要なら貼る）

関連コード：
（必要部分を貼る）

エラー全文：
（省略せず貼る）

初心者向けに、どのファイルのどこをどう直すか、手順つきで教えてください。
```

## 14. バックアップ運用ルール
初心者ほどバックアップが重要です。

### ルール
- 何か大きく直す前に、動く状態を保存する
- `App_backup_before_csv.jsx` のように一時退避でもよい
- 可能ならGitでコミットする

### 最低限の保存タイミング
- 地図が表示できた時
- 国クリックできた時
- CSV表示できた時
- 航路表示できた時
