## 5. STEP1：Node.js をインストールする

### 何のために必要か
Node.js は、React や Vite を動かすための土台です。  
これがないと `npm` コマンドが使えません。

### やること
1. Node.js の公式サイトへ行く
2. **LTS版** をダウンロードする
3. そのまま標準設定でインストールする

### インストール後の確認
Windows のターミナル、または PowerShell を開いて、次を入力します。

```bash
node -v
```

次にこれも入力します。

```bash
npm -v
```

### 成功の目安
数字が表示されれば成功です。  
例：

```bash
v22.x.x
10.x.x
```

### よくある失敗
- Node.js を入れたのにターミナルを開き直していない
- 古いバージョンが残っている
- `node` や `npm` が見つからない

その場合は、**一度ターミナルを閉じて開き直す**のが最初の対処です。

# Part B：世界地図 GeoJSON 設計書

## 17. この設計書の目的
このアプリでは、国をクリックしてニュースを表示します。  
そのため、地図データは単なる背景画像ではなく、**国ごとにクリックできる構造化データ**である必要があります。

その役目を担うのが **GeoJSON** です。

この設計書の目的は、

- どんなGeoJSONを使うべきか
- どの属性が必要か
- CSVとどうつなぐか
- どこまで最初に用意すべきか

を決めることです。

---

## 18. GeoJSONとは何か
簡単にいうと、GeoJSON は

**地図の形をJSON形式で表したデータ**

です。

このアプリでは、国ごとの境界線や面データを持たせて、クリック判定に使います。

---

## 19. このアプリでGeoJSONに必要な役割

GeoJSONには、最低でも次の役割が必要です。

1. 世界の国を表示する
2. 各国をクリックできる
3. 各国に国コードを持たせる
4. CSVの `country_iso3` と照合できる

つまり、このアプリにおいて一番重要なのは

**形そのものより、国コードが正しく入っていること**

です。

---

## 20. 最初に使うGeoJSONの条件

MVPで使うGeoJSONは、次の条件を満たしていれば十分です。

### 必須条件
- 世界地図の国単位データである
- 各国が Polygon または MultiPolygon である
- ISO3コード相当の属性がある
- React Leaflet で扱える

### 理想条件
- 国名が英語または日本語で持てる
- 不要な属性が多すぎない
- データ量が重すぎない

---

## 21. ファイル名の決め方

このプロジェクトでは、最初は次の名前で固定します。

```text
public/data/countries.geojson
```

### 理由
- 置き場所が分かりやすい
- fetchしやすい
- ChatGPTに相談するときに話が通りやすい

---

## 22. GeoJSONに欲しい属性

GeoJSONの `properties` に、最低でも次のどれかが必要です。

### 最重要属性
- `ISO_A3`
- `ADM0_A3`
- `iso_a3`
- `id`

※名前はデータによって違います。

### 表示用にあると便利
- `NAME`
- `ADMIN`
- `name`
- `name_ja`（あれば便利）

### MVP時点での最重要ポイント
属性名は何でもよいのですが、**最終的に `country_iso3` と一致させられること** が最優先です。

---

## 23. このアプリでのGeoJSON運用ルール

### ルール1：表示用の国名に頼らない
国名ではなく、必ず国コードベースで扱います。

### ルール2：GeoJSON側の国コードとCSV側の国コードを一致させる
例：
- GeoJSON：`JPN`
- CSV：`JPN`

### ルール3：最初は1ファイルだけにする
世界全体を1つの `countries.geojson` にまとめます。  
地域別に分けるのは後です。

---

## 24. 想定するGeoJSON構造

大まかには次のような構造です。

```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": {
        "ISO_A3": "JPN",
        "NAME": "Japan"
      },
      "geometry": {
        "type": "MultiPolygon",
        "coordinates": []
      }
    }
  ]
}
```

重要なのはこの2つです。

- `properties` の中にISO3コードがある
- `geometry` が国の形を持っている

---

## 25. このアプリで必要なGeoJSON処理

MVP時点で必要なのは次だけです。

1. `countries.geojson` を読み込む
2. 地図に表示する
3. 各国をクリックできるようにする
4. クリック時に `properties` からISO3コードを取る
5. CSVと照合する

### まだ不要な処理
- GeoJSON編集ツールでの細かい修正
- 地域別分割
- 最適化しすぎた軽量化
- ベクタータイル化

---

## 26. GeoJSON選定で注意すること

### 注意1：属性名は必ず確認する
GeoJSONによって国コードの属性名が違います。  
ここを確認せず進めると、クリック後にCSVとつながりません。

### 注意2：重すぎるGeoJSONは避ける
細かすぎる海岸線データは重くなります。  
最初は「見た目が十分わかる程度」でよいです。

### 注意3：例外的な地域の扱いは後でよい
台湾、コソボ、西サハラなどは政治的・データ的な扱いが難しいことがあります。  
最初は「使うGeoJSONの定義に従う」で構いません。

---

## 27. AIデータを持っている場合の考え方
あなたは平面図の AI データを持っていますが、MVPではそれを無理に最初から使わなくて大丈夫です。

### 理由
AIデータは見た目には優れていますが、クリック判定や国コード管理には向いていないことがあります。

### 結論
最初は

- 地図処理 = GeoJSON
- 見た目演出 = 後でAIデータ活用

この分担が最も安全です。

  ↓
